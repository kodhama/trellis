# Project instructions — Spec-Driven Development (Spec Kit)

This project uses **spec-driven development**. Follow this process for any change:

`constitution → specify → clarify → plan → tasks → implement`

- **Specify before code.** Write a short feature spec (`spec.md`) describing *what* and *why* before you
  write any implementation code.
- **Clarify.** If the spec has ambiguities, resolve them before planning — do not plan or build on an
  ambiguous spec.
- **Plan, then tasks.** Turn the spec into a `plan.md`, then an ordered `tasks.md`, then implement in that
  order.
- **Honor the constitution** below.

## constitution.md

1. Every feature begins as a written spec; no implementation without one.
2. Ambiguity is resolved before implementation, not during it.
3. Keep the spec, plan, and code consistent — update the spec if the design changes.
4. Small, reviewable steps; no large unreviewed changes.
