The catalog, made readable

Fourteen invariants. Why each earns its place.

These are the load-bearing properties Trellis keeps true no matter how fast your agents move. For each: what it is, why it matters, and the same situation with it and without. Everything else about your process, Trellis leaves alone.

with the invariant without it Each example is tagged by the layer it shows up in.

Structural gate

A four-point admission check: does the process even have the shape to be governed? If it lacks one, Trellis says so loudly.

inv-directional-flowDirectional flow

One-way stages of decreasing ambiguity (research → decisions → contracts → implementation → validation); downstream never consumes a draft.

Why: agents always build on settled ground — ambiguity only decreases, so no one codes against a spec that is still moving.

spec
Withoutan agent builds against a spec still being edited; it shifts, and the work is built on a version that no longer exists.
Withimplementation reads only the approved spec; a ratified doc never depends on a draft.
research
Withouta decision cites a draft finding that is later refuted — it stood on sand.
Witha synthesis cites only ratified findings, not drafts still under review.

default: enforced · independent-agent

inv-handover-pointsHandover points

Defined transitions between stages, each a place a gate can attach. (Directional flow is which way work moves; this is that it moves in steps.)

Why: development moves in discrete steps with boundaries, not one fluid blur — the seams are where work can be stopped, inspected, and handed on.

dev
Withoutvibe-coding melts prompt → code → prompt into one stream with no seam to inspect or gate.
Witha plan is "done," a spec "approved," a change "ready" — defined seams you can pause at.
CI
Withoutone monolithic script does build+test+deploy with no checkpoint to stop or roll back at.
Witheach pipeline stage is a gate a check can attach to (lint → test → build → deploy).

default: enforced · independent-agent

inv-intent-locusIntent locus

Humans own intent and values somewhere identifiable — a process with no human intent locus can't be held accountable.

Why: an accountable human owns the goal, so a wrong direction gets caught before it's built.

product
Withoutagents optimize a proxy metric no human owns, and ship the wrong thing efficiently.
Withevery feature traces to an accountable human owner; the "why" is a recorded decision.
research
Withouta research direction runs with no human sponsor and drifts somewhere no one intended.
Witha research direction has a named human sponsor who can say "that's not what we're after."

default: enforced · human (intent gate — never "nobody")

inv-ratifiable-artifactsRatifiable artifacts

Upstream can reach an approved state downstream consumes; outputs are checkable against it.

Why: you build against a stable, approved target with a real pass/fail criterion — not a moving one, and not "looks done to me."

spec
Withoutnothing is ever "final," so implementation chases a spec that keeps moving under it.
Witha spec reaches ratified, carries acceptance criteria, and work is graded against it.
data
Withoutthere is no released schema version, so services validate against inconsistent snapshots.
Witha schema is versioned; downstream validates against the approved version, not HEAD.

default: enforced · independent-agent

Operating layer

What Trellis actually supplies once a process is admissible: the gates, the checks, the memory, and the discipline that keeps agents honest.

inv-graph-maintenanceGraph maintenance

The dependency graph of artifacts and rules kept consistent and minimal; append-only records superseded, never edited in substance.

Why: the knowledge base stays coherent for the agents reading it, and a discovery deep in the code repairs the decision that should have known it (backprop).

docs
Withouta decision changes but its dependent specs are never updated — they silently diverge.
Witha repaired decision re-reviews its specs → plans → code, in turn.
research
Withouta finding contradicts an upstream note, but only the finding is recorded — the note stays wrong and agents keep reading it.
Witha downstream finding that contradicts an upstream note updates the note, not just the finding (backprop).
ops
Withoutthe same plan lives in a tracker and a wiki — the copies diverge, and the agents that should trip over the current state are reading the stale one; a parked item sits in a channel its executor never reads, trips nothing, and rots.
Witheach kind of information has exactly one home, chosen by which consumer must trip over it; recorded elsewhere first (a chat thread, a meeting note), it lands in its home before anything downstream consumes it, and the other copy just points.
code
Withouta regression test blocks a convenient reading of the spec, so it is quietly deleted — the defect it pinned ships again; no one can say which requirement any test guards.
Withevery test names the upstream it guards (a spec anchor or a defect id); when a test and the spec disagree, the conflict is surfaced and resolved deliberately — the spec gains its missing invariant, or the over-pinning test is retired, citing why. A regression test is never weakened just to make a reading of the spec pass.

default: enforced · independent-agent

inv-gate-at-handoverGate at handover

Apply the verification gate at every handover; any skip is surfaced.

Why: the review actually fires (not quietly skipped under deadline) — and if it is skipped, you can see it was.

CI
Withoutthe review is "optional," so under deadline it silently doesn't happen and a defect ships.
Witha conformance check + review fire on every PR; a deliberate skip is recorded, not hidden.
release
Withoutan artifact is promoted straight to prod with no gate, and no record the check was skipped.
Witha promotion gate blocks anything that didn't pass the prior stage's sign-off.

default: default-on-but-skippable · independent-agent

inv-independent-judgmentIndependent judgment

The assessor is independent of what it assesses — the builder doesn't grade itself, and the agent doesn't flatter the human.

Why: the builder doesn't grade its own homework, and the agent names the risk instead of flattering the plan — so verification and the intent gate are real, not decorative.

review
Withoutthe agent that wrote the code reviews its own code and decides it's good.
Witha read-only reviewer, distinct from the author, derives its own checklist and reports what's wrong — even when inconvenient.
research
Withoutan agent certifies its own findings (or agrees with a flawed plan to please you), and it sails through.
Withfindings are adversarially verified by a separate pass, not self-certified.
collab
Withoutthe human suggests the old code is wrong; the agent answers "you're right, good catch" without opening it — the code was fine, and the plausible agreement ships a bug.
Withbefore agreeing or disagreeing with the human's hunch — especially when the verdict would match it — the agent opens the file, quotes the actual lines, runs the obvious counter-checks, and labels what is fact and what is inference; "I can't confirm this" is said out loud when it can't.

default: default-on-but-skippable · independent-agent

inv-auditable-archiveAuditable archive

Provenance + immutable decision history + a consolidated current-truth.

Why: you can always answer "why is it this way?" — decisions are not lost or quietly rewritten.

ADR
Withouta decision is edited in place and its rationale lost; months later it is re-litigated from scratch.
Withdecisions are append-only and link their rationale; superseding writes a new record.
infra
Withoutan undocumented prod change, so no one can say why it is configured this way.
Withevery prod change carries provenance — git history + a current-truth doc — so "why" is always answerable.

default: enforced · independent-agent

inv-bounded-contextBounded context

Each operation reads only its declared inputs, never the whole archive.

Why: agents decide on sharp, relevant context instead of drowning in everything — better calls, and it scales as the archive grows.

agent
Withoutan op dumps the entire repo into context, dilutes the signal, and decides on noise.
Witha sub-agent gets exactly its declared inputs and decides crisply within them.
data
Withouta query scans the whole warehouse, drowns in irrelevant rows, and gets slower as it grows.
Witha query reads a scoped view, not the whole warehouse.

default: default-on-but-skippable · independent-agent

inv-self-improvementSelf-improvement

The process learns from friction and gets better — improvement signals are surfaced and acted on, so a glitch doesn't happen twice.

Why: a process glitch never happens twice — friction becomes a fix, not a recurring tax.

CI
Withoutthe same pipeline step fails weekly and everyone just re-runs it, forever.
Witha flaky test recurs → a trigger is filed, the root cause fixed, the trigger retired in the same change.
process
Withouta PR raises the same open question every time, with no follow-up, and it rots unowned.
Witha repeated review miss becomes a checklist item that rides the PR you already write.

default: default-on-but-skippable · human

inv-minimal-firstMinimal first

The smallest process that works; add a step only when friction reveals the boundary; bias to retire over add.

Why: no ceremony for its own sake — every step has earned its place, so the process stays light.

process
Withouta heavyweight methodology copied wholesale, most steps cargo-culted.
Witha deliberately tiny rule set; a new step lands only with a recorded reason.
tooling
Withouta whole framework pulled in to use one function; the build carries steps no one remembers.
Witha build config with no unused steps; a dependency pulled in only when needed.

default: expressed · human

inv-clarify-before-commitClarify before commit

Ambiguity in an upstream is surfaced and resolved (usually by asking a human) before downstream consumes it — never silently guessed.

Why: agents ask instead of guessing — you don't discover three files later that they took the wrong reading of a vague spec.

spec
Withoutan agent silently picks one reading of a vague spec, builds it, and it's the wrong one.
Withan agent flags a vague requirement and resolves it before coding.
data
Withoutan ambiguous metric definition is guessed, and the dashboard is subtly, confidently wrong.
Withan ambiguous metric definition is confirmed with a human before the dashboard is built.

default: default-on-but-skippable · human

The two floors

The only settings that never dial to zero. Everything else Trellis lets you tune; these two it holds firm.

floor-transparencyTransparency

Every consequential choice is surfaced — a skipped gate, a missing capability, a degraded result, a divergence from a framework you claim to follow. Drift is allowed, but never silent.

Why: nothing consequential happens silently — you learn about the shortcut when it's taken, not when it breaks in production.

framework
Withoutthe team quietly drifts from the methodology it claims to follow.
With"we diverge from Spec Kit here" is a recorded decision, so an agent knows where you follow the book and where you don't.
ops
Withoutan agent silently falls back to a degraded path or swallows an error — you learn when it breaks in prod.
Witha degraded fallback (cache miss, retry, downgrade) is surfaced, not swallowed.
agent
Withouta run dies at its turn cap mid-task and nothing marks where it stopped — the next run redoes the work from zero, or the half-finished state sits unnoticed until something breaks.
Withevery bounded run leaves resumable state — the work pushed, a note of what's left — so a successor continues instead of restarting; auto-resumes are bounded, and hitting the bound produces a loud demand for human attention, never a quiet dead-end.

floor: enforced (non-configurable to off) · human

floor-intent-gateThe intent gate

The intent gate never fully opens — at the intent locus, a human (or a human-authorized independent check) is always mandatory. The one place an upstream that is itself wrong gets caught.

Why: "is this the right thing to build?" always has a human behind it — the one call you never hand fully to the machine.

product
Withouta fully-automated pipeline ships something technically correct that no human confirmed was the right thing.
Witha human ratifies at the intent gate — the approval recorded as a human intent act, whatever mechanism performs it.
release
Withouta deploy auto-promotes a change no human approved, on green tests alone.
Withno deploy ships a feature no human approved, however green the pipeline.

floor: enforced (non-configurable to off) · human (never "nobody")
